    inferlite#

    从零手撕的 LLM 推理引擎学习项目 · L0-aligned with transformers

     立即开始  查看进度  GitHub

    文档导航#

    1-plan/14 个里程碑路线图、当前作战地图 M1、整体进度跟踪。PLAN · PROGRESS · M<n>

    2-tasks/任务卡七字段（前置 / 边界 / 验收 / 风险 / 完成总结），一卡一文件。M1-T0 · M1-T1 ✓ · M1-T2

    3-kb/知识库：论文 / 库 / 概念 / 工具 卡片化总结，踩坑教训 + ADR 决策记录。knowledge · lessons · decisions

    4-setup.md一键 uv install、常用 make 命令、仓库结构速查、大坑预警。uv · make · ruff · pytest

    三类文档#

    分类 时间维度 写作时机 文件
    1-plan/ 未来 / 当前 规划 + 持续更新 PLAN.md PROGRESS.md M<n>.md
    2-tasks/ 当前 开工时写、完成时收尾 M<n>-T<x>-*.md
    3-kb/ 过去（沉淀） 任务结束后归档 knowledge.md lessons.md decisions.md REFERENCES.md
    4-setup.md 长期常驻 项目稳定后不常变 4-setup.md

    三条阅读路径#

    1. 你正在看的 README.md — 知道有哪些文档
    2. 4-setup.md — 5 分钟跑起项目
    3. 1-plan/PLAN.md — 看整体路线
    4. 3-kb/decisions.md — 理解为什么这么做
    1. 1-plan/PROGRESS.md — 找下一张 任务
    2. 2-tasks/ 中读对应任务卡 7 字段
    3. 3-kb/knowledge.md — 按"前置"段查相关章节
    1. 3-kb/lessons.md — 全部踩坑
    2. 当前 M<n>.md 末尾 Summary
    3. 任务卡末尾"完成总结"段
